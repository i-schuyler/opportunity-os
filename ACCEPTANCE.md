# ACCEPTANCE.md

## Opportunity OS v0.1 acceptance

A change is acceptable only if all relevant items below are true.

### Product
1. A new user can sign in.
2. A signed-in user can create an opportunity.
3. A user can edit, archive, and view opportunity records.
4. Each opportunity supports title, type, source link, contact, deadline, status, tags, and notes.
5. A user can save and reuse a response template.
6. A paid user can access premium-gated features.
7. A scheduled weekly summary path exists and is testable.

### Delivery
8. CI passes via `./ci.sh`.
9. PR template fields are present and valid.
10. Workflow-sensitive changes do not auto-merge.

### Business
11. Marketing copy presents Opportunity OS for opportunity seekers generally, with housing as the lead example.
12. Founder lifetime pricing is described as limited to the first 50 customers.
